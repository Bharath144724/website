# Automated WordPress Deployment with Nginx and GitHub Actions This repository contains the code and configurations for automating the deployment of a WordPress website using Nginx as the web server, a LEMP (Linux, Nginx, MySQL, PHP) stack, and GitHub Actions for CI/CD automation. ## Table of Contents - Prerequisites - Server Provisioning - LEMP Stack Installation - WordPress Installation and Configuration - GitHub Repository and Actions Setup ## Prerequisites Before you begin, make sure you have the following prerequisites in place: - A virtual private server (VPS) (AWS) with a secure Linux distribution (eg: amazon linux 2023) provisioned. - Domain name or IP address pointing to your server. - GitHub account for version control and CI/CD automation. ## Server Provisioning 1. Provision a VPS with your preferred cloud provider(ex:Azure, AWS). 2. Set up SSH key-based authentication for secure access. ## LEMP Stack Installation 1. Install Nginx, MySQL, and PHP on the server. 2. Configure MySQL with secure settings. 3. Create a MySQL database for WordPress. ## WordPress Installation and Configuration 1. Download and set up WordPress on the server. 2. Configure WordPress database settings. 3. Harden WordPress security with strong passwords and limited user privileges. ## GitHub Repository and Actions Setup 1. Create a GitHub repository for your WordPress project. 2. Set up a GitHub Actions workflow for automated deployment. 3. Define necessary workflow steps, including dependency installation and file transfer. ## Usage To deploy your WordPress website: 1. Clone this repository to your local machine. 2. Follow the instructions in the [GitHub Actions workflow](.github/workflows/main.yml) for deployment. 3. Commit your changes and push to the repository.
